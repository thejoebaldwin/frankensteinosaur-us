Crypto LabelMaker – Computer Security

This was a client/server program involving a password-sharing scheme. The key required to digitally sign a message could only be constructed when three of the generated passwords were present.

     The passwords were generated using a threshold scheme (A. Shamir, “How to share a secret” Communications of the ACM ,v. 24, n.11 Nov 1979). We wish for the key to be constructed using three passwords so we used a polynomial of degree 2 (3-1 = 2).

                               password = f(x) = (ax^2 +bx + M) mod P

P a prime, M the key, and a and b, are all chosen at random. P is kept, M, a and b are discarded afterwards. Evaluating the polynomial at different points generates “partial” passwords. To reconstruct the key, M, the set of 3 equations were solved through polynomial interpolation to give back a and b, in which then M could be determined.

     After the passwords are generated initially, a public key (server side) would also be generated, and distributed along with the individual passwords. To reconstruct the key, each user would run the client program and enter their password. When three keys were entered, the original key would be reconstructed. An arbitrary message would be encrypted using an RSA scheme with the private key and then would be used as a digital signature on a document. Should the authenticity of the signature come into question, the public key would be used to decrypt it and the result would match the original message.
